# Beyond Distributional Similarity: Evaluating Predictive Utility of Synthetic Liver Cirrhosis Data

**Authors:** Michael Udousoro et al.
**Dataset:** Mayo Clinic Primary Biliary Cirrhosis (PBC) Trial — [`cirrhosis.csv`](data/raw/cirrhosis.csv)
**Status:** Research pipeline — article in preparation

---

## Overview

Primary Biliary Cirrhosis (PBC) is a rare autoimmune liver disease. The Mayo Clinic trial
enrolled only 312 randomised patients, and after removing records with missing values the
usable dataset contains 276 complete cases. This scarcity makes reliable machine learning
model training difficult.

This project investigates whether synthetic data augmentation can improve predictive models
trained on this limited real data. Three deep generative models are compared — a Vanilla GAN,
a Conditional Tabular GAN (CTGAN), and a Tabular Variational Autoencoder (TVAE) — all built
from scratch using TensorFlow 2.15 without any third-party synthetic data library.

The pipeline goes beyond standard distributional fidelity evaluation (FID, KS tests) and
asks the clinically meaningful question: does adding synthetic patients to the training set
actually improve a model's ability to predict whether a real patient will survive?

### Three methodological contributions

1. **IQR-based outlier filtering** — removes physiologically implausible synthetic records
   before they enter any downstream analysis, using bounds derived from real patient data.

2. **Multi-method consensus voting** — identifies a high-confidence subset of synthetic
   records that all three generative models independently agree are realistic.

3. **Predictive utility evaluation** — compares model performance across three training
   scenarios (real only, real + CTGAN, real + consensus) on a strictly held-out real test set.

## Dataset

**Source:** [UCI Machine Learning Repository — Cirrhosis Patient Survival Prediction](https://archive.ics.uci.edu/dataset/878/cirrhosis+patient+survival-prediction+dataset)

| Property | Value |
|---|---|
| Raw rows | 418 patients |
| Complete cases used | 276 patients |
| Rows dropped (any missing value) | 142 |
| Features | 18 (11 continuous, 5 binary, 2 ordinal) |
| Target | Status — D=1 (died), C/CL=0 (survived or transplant) |
| Train split | 193 patients (70%, stratified) |
| Test split | 83 patients (30%, held out, never used in training) |
| Class balance (train) | 115 survived / transplant, 78 died |

### Why 142 rows are missing

Patients 1–312 were part of the randomised controlled trial with full clinical measurements.
Patients 313–418 were an additional observational cohort — most clinical lab values were not
collected for them. This explains why 91 rows are missing exactly 9 columns and 15 rows are
missing exactly 10 columns. Complete-case analysis was chosen because imputing values for
an observational cohort collected under fundamentally different conditions would be
scientifically questionable.

## Notebooks — Run in Order

### [`00_exploratory_data_analysis.ipynb`](00_exploratory_data_analysis.ipynb)

Full data exploration of the 276 complete cases:

- Descriptive statistics and outcome-stratified means
- Missing value heatmap
- Side-by-side histograms with KDE curves (survived vs died)
- Box plots and violin plots for all 11 continuous features
- Pearson correlation matrix
- Pairplot of six key clinical features
- IQR outlier profile
- Shapiro-Wilk normality tests
- **3D Clinical Biomarker Space** — Bilirubin × Albumin × Prothrombin, 4 viewing angles
- **3D PCA Projection** — all 11 continuous features reduced to 3 components, loading arrows
- **3D KDE Density Surfaces** — Bilirubin × Prothrombin probability surfaces by outcome
- **3D Bubble Plot** — 4-dimensional view with N_Days encoded as bubble size

### [`01_complete_case_data_preparation.ipynb`](01_complete_case_data_preparation.ipynb)

- Loads raw CSV and drops patient ID
- Applies complete-case filter (276 rows retained)
- Encodes all categorical and ordinal features to numeric
- Fits MinMaxScaler on training data only
- Performs stratified 70/30 split (random seed 42)
- Saves `train_real.csv`, `test_real.csv`, `scaler.pkl`

### [`02_synthetic_generation_iqr_filtering_and_consensus.ipynb`](02_synthetic_generation_iqr_filtering_and_consensus.ipynb)

- Trains Vanilla GAN, CTGAN, and TVAE on scaled training data
- Generates 500 synthetic records per method
- Applies IQR-based Tukey fence filter to remove implausible records
- Runs multi-method consensus voting (tolerance 0.5, min votes 2)
- Computes tabular FID scores for all synthetic datasets
- Runs five statistical tests: Shapiro-Wilk, KS, JSD, Cohen's d, chi-square
- **t-SNE visualisation** — 2D projection of real vs synthetic in 18-dimensional feature space
- **Privacy disclosure risk analysis** — nearest-neighbour distance approach

### [`03_predictive_modeling_and_statistical_evaluation.ipynb`](03_predictive_modeling_and_statistical_evaluation.ipynb)

- Trains Random Forest, Gradient Boosting, Logistic Regression under 3 scenarios
- **Scenario A** — 193 real patients only (baseline)
- **Scenario B** — real + filtered CTGAN (best FID score)
- **Scenario C** — real + consensus synthetic
- **Scenario D** — real + SMOTE oversampling (classical baseline)
- 5-fold stratified cross-validation with AUC mean and standard deviation
- Bootstrap confidence intervals (1000 resamples) for test-set AUC
- McNemar's exact test comparing augmented vs baseline scenarios
- ROC curves and Precision-Recall curves for all scenario/classifier combinations
- Feature importance from Random Forest and Gradient Boosting
- Full plain-text summary report

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/michaeludousoro/liver-cirrhosis-synthetic-data.git
cd liver-cirrhosis-synthetic-data
```

### 2. Create a virtual environment and install dependencies

**macOS / Linux**
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**Windows (Command Prompt)**
```cmd
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

**Windows (PowerShell)**
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

> **Windows PowerShell note:** If you see an error about running scripts being disabled,
> run `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser` once and
> then retry the activate command.

### 3. Register the kernel with Jupyter
> Skip this step if you are using VS Code — it detects the virtual environment automatically.


**macOS / Linux**
```bash
python3 -m ipykernel install --user --name=liver_cirrhosis_article \
        --display-name "Python (liver cirrhosis article)"
```

**Windows**
```cmd
python -m ipykernel install --user --name=liver_cirrhosis_article --display-name "Python (liver cirrhosis article)"
```

### 4. Run notebooks in order

Open VS Code (or JupyterLab) in the project root and run each notebook from top to bottom.
Select the `Python (liver cirrhosis article)` kernel.

Notebooks must be run in order — each saves outputs that the next one loads:

```
00  →  (no dependencies — standalone EDA)
01  →  saves train_real.csv, test_real.csv, scaler.pkl
02  →  requires outputs from 01
03  →  requires outputs from 01 and 02
```

## Reproducibility

All random seeds are fixed at 42 throughout the pipeline — the train/test split,
all generative models, all classifiers, and all resampling procedures.
Running the notebooks on the same hardware and library versions will produce
identical numerical results.

> Note: TensorFlow operations on GPU hardware are not fully deterministic even with
> a fixed seed. Small numerical differences are possible on GPU machines. The results
> reported in the paper were produced on Apple Silicon (M-series) CPU.

## Key Results

The table below shows headline performance numbers for the best-performing classifier
(Gradient Boosting) across all four training scenarios. Full results including 95%
bootstrap confidence intervals, 5-fold cross-validation AUC, and McNemar test p-values
are in [`output/results/summary_report.txt`](output/results/summary_report.txt).

|---|---|---|---|---|
| A: Baseline (real only) | Gradient Boosting | 0.7831 | 0.7852 | 0.8776 |
| B: Real + CTGAN | Gradient Boosting | 0.7470 | 0.7497 | 0.8552 |
| C: Real + Consensus | Gradient Boosting | 0.7831 | 0.7848 | 0.8836 |
| D: Real + SMOTE | Gradient Boosting | 0.8072 | 0.8087 | 0.8661 |
